## Blockers / risks

- Production hardening still requires cross-device Apple Silicon sign-off (M4 tier spread) and CI envelope calibration beyond this M1 profile before claiming globally optimized default knobs across hardware classes.
- Strict consensus safety for Apple Metal now relies on candidate-only CPU confirmation for solved headers in strict validation mode (instead of forcing strict mining backend to CPU), preserving deterministic acceptance while keeping Metal solve active.
- BTX default functional matrix (`22` scripts) is now green, but overall coverage remains intentionally narrower than upstream base coverage (`357` scripts) unless explicit opt-in is used (`BTX_RUN_UPSTREAM_BASE_SUITE=1` or `--extended`).
- Non-wallet chain hardening (post-genesis `nMinimumChainWork` + checkpoint anchors) now has deterministic generate/apply/check automation (`scripts/update_chain_hardening_manifest.py`, `scripts/apply_chain_hardening_manifest.py`) and util test coverage; remaining work is operational data collection from canonical mainnet peers before running the finalized release patch.
- Qt + documentation surface area is broad; strict string tests may reveal additional downstream updates.
- BTX fork policy/address constraints differ from upstream assumptions; wallet/mempool scripts now run deterministic BTX smoke assertions in `p2mr`/`scriptpubkey` modes (`wallet_fundrawtransaction.py`, `rpc_psbt.py`, `mempool_accept.py`, `feature_rbf.py`, `rpc_rawtransaction.py`) instead of skipping.
- KAWPOW mixhash-specific rejection checks are conditional when `fKAWPOW` fields are disabled (`feature_btx_kawpow_consensus.py`), while extended-header serialization and reduced-data consensus checks remain active.
- On macOS Qt test harness with `QT_QPA_PLATFORM=minimal`, `AppTests`/`WalletTests`/`AddressBookTests` are intentionally skipped by upstream test guards.
