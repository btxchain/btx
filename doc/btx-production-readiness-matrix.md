# BTX Production Readiness Matrix

This matrix gives operators a single view of every BTX verification path, the
milestone it unblocks, how to re-run it, and what is still outstanding. Keep it
close to the code that enforces those guarantees (`scripts/`, `test/`, and
`doc/`) so the table always mirrors reality.

For shielded launch decisions, read this matrix together with
`doc/btx-shielded-production-status-2026-03-20.md` and
`doc/btx-smile-v2-genesis-readiness-tracker-2026-03-20.md`, plus the stable
security index in `doc/security/README.md`. This matrix covers
repeatable repo-side verification commands, but it does not override the later
reset-chain launch sign-off in those March 20-23 source-of-truth documents.

**Legend**

- `Green`: verification is repeatable with the commands shown.
- `Yellow`: verification path exists but has open follow-ups.
- `Red`: verification blocked on upstream data or code.
