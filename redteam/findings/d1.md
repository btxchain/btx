## Post-base CVE sweep (G2) — are upstream fixes present?
Base is the 2025-11-10 Knots snapshot, so only advisories AFTER that matter. The relevant recent
Core advisories (disclosed Oct 2025, fixed in v30.0) are already in this tree:
- **CVE-2025-54605** (disk-fill via invalid-block logging) — FIXED. `LogRateLimiter` class present
  (src/logging.h:115; the PR #32604 log rate-limiter).
- **CVE-2025-54604** (disk-fill via spoofed self-connection logging) — FIXED. self-connect log is
  now `LogDebug` with the explicit "avoid log amplification" comment (net_processing.cpp:4260-4263).
- **CVE-2024-52911** (UAF in script interpreter via crafted block) — fixed upstream in 29.0; base
  is 29.2 ⇒ included.
- **CVE-2018-17145 / INVDoS** (orphan memory) — mitigated: modern weight-based TxOrphanage with
  per-peer usage caps + LimitOrphans (txorphanage.{h,cpp}); MAX_STANDARD_TX_WEIGHT bound on entry.
- Residual: CVE-2025-46597 (32-bit-only remote crash) and CVE-2025-46598 (mempool resource use)
  were also v30.0 fixes; I did not byte-verify these two specific patches. 32-bit is irrelevant to
  the deployed 64-bit nodes; the mempool-resource one is LOW. → see open question.

## Vectors attempted (empirical, regtest `rtD1-node` on image btx-node:pr214, namespace rtD1)
- **CVE-2018-17144 duplicate-input inflation**: createrawtransaction listing the SAME 20-BTC
  outpoint TWICE, output 39.999 BTC, wallet-signed `complete:true`.
  RESULT: REJECTED. testmempoolaccept → `bad-txns-inputs-duplicate`; sendrawtransaction → err -26
  `bad-txns-inputs-duplicate`; generateblock (force into mined block, bypasses mempool) →
  "ProcessNewBlock, block not accepted". No inflation, no assert-crash.
  CHECK: src/consensus/tx_check.cpp:66-70 (`std::set<COutPoint> vInOutPoints` dup check) — INTACT
  after the shielded refactor; CheckTransaction is still called from CheckBlock for every tx.
- **CVE-2010-5139 value-sum overflow / MoneyRange**: two outputs of 15,000,000 BTC each (each
  within per-output MAX_MONEY, sum 30M > MAX_MONEY).
  RESULT: REJECTED. testmempoolaccept → `bad-txns-txouttotal-toolarge`; generateblock → not accepted.
  Single output > MAX_MONEY caught even earlier at RPC encode (`Amount out of range`).
  CHECK: tx_check.cpp:48-59 running-sum `nValueOut += ...; if(!MoneyRange(nValueOut))`. INTACT.
- **Shielded↔transparent value boundary (the fork-specific inflation surface)**: code audit of
  Consensus::CheckTxInputs (tx_verify.cpp:169-229). value_balance from GetShieldedTxValueBalance is
  range-checked `MoneyRangeSigned` (206), folded via `CheckedAdd(nValueIn,value_balance)`+MoneyRange
  (210-213), and `adjusted_value_in < value_out` enforces no inflation (216). GetValueOut throws on
  overflow (primitives/transaction.cpp). Raw bundle value_balance/fee returned unguarded by
  GetShieldedTxValueBalance (bundle.cpp:1091) but EVERY consensus call site bounds it ⇒ safe.
  RESULT (code-reasoned): robust; uses modern overflow-safe CheckedAdd throughout. No empirical
  shielded-tx mint harness built (no easy CLI to craft an over-balance bundle) — flagged to c5.
- **CVE-2012-2459 merkle duplicate-txid**: generateblock with one valid tx listed TWICE → block
  built (nTx=3, height 102) but `confirmations:-1`, tip stayed at 101 ⇒ REJECTED, never connected
  (this instance tripped `bad-txns-inputs-missingorspent` on the 2nd copy's already-spent input).
  RESULT: REJECTED (no fork/inflation). The dedicated mutation defense is INTACT by inspection:
  ComputeMerkleRoot sets `mutated` on equal adjacent leaves (merkle.cpp:46-66); CheckMerkleRoot
  rejects BLOCK_MUTATED `bad-txns-duplicate` (validation.cpp:8022-8027); plus IsBlockMutated +
  64-byte-tx defense (validation.cpp:8273+) and the `m_check_block_mutated_mock` hook. Net layer
  bans peers sending mutated blocks (net_processing.cpp:5919-5922). I did NOT hand-craft a block
  whose duplicated leaves land exactly on an odd-level boundary + submitblock — left as residual.
- **CVE-2012-1909 / BIP30 dup-coinbase**: code present — IsBIP30Repeat / fEnforceBIP30
  (validation.cpp:5733-5749), with the known mainnet exception heights. INTACT.
- **BIP34** height-in-coinbase: GetBlockScriptFlags (validation.cpp:5588). **BIP66** strict-DER:
  SCRIPT_VERIFY_DERSIG (5607-5609); LOW_S / STRICTENC flags (4148-4164). INTACT.
- **Subsidy/halving off-by-one**: GetBlockSubsidy (validation.cpp:4901) — `halvings=nHeight/interval`,
  `if(halvings>=64) return 0` undefined-shift guard, `nSubsidy>>=halvings`. Uses nInitialSubsidy
  param (not hardcoded 50). Logic identical to upstream; no off-by-one. (Not separately repro'd.)
- **Coinbase maturity**: spent a depth-1 coinbase via generateblock → REJECTED
  `bad-txns-premature-spend-of-coinbase` (tx_verify.cpp:187, COINBASE_MATURITY=100). ENFORCED.
