# u2 — Startup/rebuild progress reporting + ETA (UX / observability, non-consensus)

## Summary
Implemented periodic progress reporting (percent + textual progress bar + rate +
**ETA**) for the long, previously-silent startup/rebuild phases, plus a small
reusable progress/ETA helper. Target was the shielded-notes genesis→tip replay
(`RebuildShieldedState`) that ran **~25 minutes silently** on the production node
(only a start + end line). Same helper is reused for reindex and IBD, the
shielded-rebuild phase is announced up front via `InitMessage`, and live
rebuild progress is exposed over RPC in `getblockchaininfo`.

All changes are **non-consensus** (pure logging / RPC status). Built clean
(Alpine/CMake via the repo Dockerfile, `btx-node:rtU2`) and verified in regtest.

## What changed
1. **New reusable helper** `src/util/progress.h` — header-only `ProgressReporter`:
   - `MaybeReport(done)` → throttled `std::optional<std::string>` (default every
     10 s; reindex uses 5 s) so it never spams steady-state.
   - `Final(done)` → always-rendered 100% summary line.
   - Renders: `[label] NN.N% [####....] done/total  rate/s  elapsed Xs  ETA Ys`.
     ETA = elapsed · (1−frac)/frac from current rate; durations format as
     `s` / `m s` / `h m s`. Steady (mockable-in-tests) clock; clamps to [0,1].
2. **Shielded-notes rebuild** (`RebuildShieldedState`, validation.cpp): added a
   start line (block count + height range) and a throttled progress bar/ETA
   over the genesis→tip replay loop; publishes coarse progress for RPC; final
   summary line. Closes the ~25-min black box.
3. **Reindex** (`ImportBlocks`, node/blockstorage.cpp): counts block files up
   front, then reports percent/ETA/bar per file (reuses the same helper).
4. **IBD / sync** (`Chainstate::UpdateTip`, validation.cpp): throttled
   `[Syncing blockchain] NN.N% [...] height ETA` line driven by
   `GuessVerificationProgress`; clears on leaving IBD.
5. **Verbose phase logging on by default** (init.cpp): the shielded init phase
   is now announced with `uiInterface.InitMessage(_("Rebuilding shielded state…"))`
   (logged as `init message: …` by noui). The other major phases — "Loading
   block index…", "Verifying blocks…", "Loading wallet…", "Done loading" — were
   already InitMessage-logged; this fills the one gap before the long replay.
   `EnsureShieldedStateInitialized` already logs which path it takes (restore vs
   full rebuild). No new steady-state spam.
6. **RPC** (`getblockchaininfo`, rpc/blockchain.cpp): optional `rebuild_progress`
   object `{phase, progress, done, total}`, present only while a rebuild/reindex/
   sync phase is active. Read via a dedicated mutex (no cs_main contention).

`ChainstateManager` gains `StartupProgress` + `Set/GetStartupProgress()` (its own
`Mutex`, not `cs_main`) so any phase can publish and the RPC thread can read cheaply.

## Notes / scope
- Strictly non-consensus: logging + an optional RPC status field. No change to
  validation, PoW, ASERT, shielded crypto, or block acceptance.
- Throttle keeps steady-state quiet; only long startup/rebuild phases emit.
- Minor cosmetic: when a phase finishes within one throttle window, the last
  `MaybeReport` and `Final` can print one duplicate 100% line (seen in the 1/1
  reindex). Harmless; left as-is since multi-file/large replays differ.
- Safety: developed in /tmp/rtU2-src (git archive pr-214), built/tested only in
  the `rtU2-` namespace on regtest; production btxd.service/.btx/.local/bin and
  all pf3-*/bft-*/evx*/btxn*/other rt* untouched; all rtU2 containers/images and
  /tmp scratch cleaned up.

+        m_last_report = now;
+        return Render(done, now, /*final=*/false);
+    }

+    /** Always render the final summary line (100% / total elapsed). */
+    std::string Final(int64_t done)
+    {
+        const auto now{SteadyClock::now()};
+        return Render(done, now, /*final=*/true);
+    }
+

+private:
+    std::string Render(int64_t done, SteadyClock::time_point now, bool final) const

+        } else {

+        }
