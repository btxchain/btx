# e1 — MatMul PoW *proof-of-WORK* break: per-nonce O(n³)→O(n²r) amortization (mine cheaper than honest)

Scope (orthogonal to b1, which refuted *forgery* of a fake C'): can a miner produce a
**genuine, fully-valid** block — correct C' = A'·B', correct product-committed digest,
passes Freivalds — while doing **materially less field work per nonce than the honest
Ω(n³) cost**, i.e. break the proof-of-WORK property and out-mine honest miners?

**YES. CRIT.** The dense O(n³) part of every nonce attempt is *identical across the entire
grind* and can be precomputed ONCE per chain tip; per-nonce work then drops to O(n²·r).
Measured (numpy field-exact model, digest bit-identical to honest): **per-nonce field-mult
speedup 12.8× on mainnet (n=512, r=8), 3.4× on regtest (n=64, r=4)**; block-level ≈12.8×.
The produced digest/C' are bit-identical to the honest miner's, so the attack is
**completely undetectable by consensus** — these blocks are indistinguishable from honest ones.

## Why it works (the algebra)

Per nonce the prover must compute, for each output block (i,j) (mainnet product-committed
scheme, `ComputeProductCommittedDigestFromPerturbed`, transcript.cpp:511):
```
acc[i,j] = Compress_block(i,j)( sum_ell A'_block(i,ell)·B'_block(ell,j) )
         = Compress_block(i,j)( (A'B')_block(i,j) )
```
with A' = A + E, B' = B + F, E = E_L·E_R (rank r), F = F_L·F_R (rank r), and `Compress` a
**linear** form (dot with the sigma-derived vector v). Expand A'B':
```
A'B' = A·B  +  A·F + E·B + E·F   ( = AB + Mnoise )
```
Decisive facts:
- **A and B are FIXED for the whole grind.** seed_a/seed_b = `DeterministicMatMulSeed(prev_block_hash, height, which)` = `SHA256(prev||height||which)` (pow.cpp:47), enforced in consensus (validation.cpp:8385). They depend ONLY on (prevhash, height) — NOT on nonce, time, merkle root, or extranonce. ⇒ `A·B` is the same matrix for **every** nonce attempt at a given tip.
- During mining only `nNonce64` varies per attempt (BuildMatMulNonceBatchWindow, pow.cpp:1069-1072); everything feeding seeds is constant. ⇒ AB is amortizable across the ENTIRE block-mining epoch, not just one template.
- `Compress` is linear ⇒ `acc[i,j] = Compress(AB_block(i,j)) + Compress(Mnoise_block(i,j))`.
- `Mnoise = A·F + E·B + E·F` is GLOBALLY rank ≤ 3r and computable by associativity in O(n²r):
  `A·F = (A·F_L)·F_R`, `E·B = E_L·(E_R·B)`, `E·F = E_L·((E_R·F_L)·F_R)`.

So the attacker:
- **ONCE per tip:** `AB = A·B`  (a single dense O(n³) product).
- **PER NONCE:** sigma → noise (O(n²r)) → `Mnoise` global low-rank (O(n²r)) → add to AB →
  `Compress` all N² blocks (O(n²)). **No per-nonce dense product.** Full C' = AB+Mnoise is
  also available in O(n²r), so the Freivalds C' payload is filled cheaply too.

The honest miner (consensus reference path, accelerated_solver.cpp:34-50 →
`ComputeProductCommittedDigestFromPerturbed`) recomputes the **dense N³ block-products
`A'_block·B'_block` per nonce** — O(n³) every attempt. That dense work is exactly the
sigma-independent `A·B` re-done every time. **That is the wasted, amortizable work.**

## Empirical confirmation (field-exact numpy model, p=2^31−1)

`/tmp/rtE1/optimal2.py` builds A,B,E,F,v over F_p and checks the amortized digest ==
honest digest for several (n,b,r); ALL PASS (n=16/32/48/64). Op-counts (field mults):

| net | n,b,r | honest/nonce | optimal/nonce | precompute (once) | **per-nonce speedup** |
|-----|-------|-------------:|--------------:|------------------:|----------------------:|
| regtest | 64,8,4 | 299,008 | 88,064 | 262,144 | **3.4×** |
| mainnet | 512,16,8 | 138,674,176 | 10,813,440 | 134,217,728 | **12.8×** |

Block-level (mainnet, eps=18 ⇒ ~2^18 matmuls per target hit, `/tmp/rtE1/epsilon.py`):
honest ≈3.64e13 mults/block vs optimal ≈2.83e12 ⇒ **12.8× cheaper per block at the same
difficulty.** The O(n³) precompute is paid once and is <5% of one block's honest cost, so it
amortizes to ~nothing. The legacy/pre-61000 transcript scheme (prefix-hashed) amortizes
less but still gives **5× on mainnet** (floor O(n³/b); `/tmp/rtE1/legacy.py`).

Asymptotic: honest = Θ(n³), optimal = Θ(n²r) ⇒ **speedup ≈ n/(6r)**. Grows with n; the design's
"Ω(n³) per attempt" claim is false — true per-attempt cost is **Θ(n²r)** after one Θ(n³) setup.

## Relation to b1 (does NOT overlap)
b1 refuted producing a *fake* C'≠A·B (Fiat-Shamir binds r to C'); b1 also stated "AB is
irreducibly dense O(n³)". That is true **for a single isolated nonce** but b1 missed that AB
is **identical across all nonces** (only v and the rank-r noise change), so the O(n³) is paid
once, not per attempt. This attack produces the *honest* C' = A·B (so Freivalds/digest pass
trivially) — it attacks WORK, not soundness. Different vector, both valid.

## Vectors attempted
- **Naive block-wise reuse** (`ReplayCanonicalHashWithReusableCleanProducts`, the dev's own
  `bench/matmul_reuse_bench.cpp`): precompute clean block products, redo low-rank corrections
  per block-triple. RESULT: **0.34× (3× SLOWER)** — the per-(i,j,ell) corrections cost 5b²r+2br²
  ≈ b³, losing. This is why the shipped reuse bench shows reuse "doesn't help" — it measures the
  WRONG (block-local) decomposition. (`/tmp/rtE1/cost_model.py`, `field_model.py`.)
- **Global low-rank amortization** (compute Mnoise once globally via associativity, not per
  block): RESULT: **3.4× / 12.8× faster, digest bit-identical.** The shipped reuse path is a
  red herring; the real win needs the global-rank collapse. (`/tmp/rtE1/optimal2.py`.)
- **Seed/nonce dependence check:** seeds = f(prevhash,height) only, consensus-enforced ⇒ A,B
  fixed across grind ⇒ amortization valid. CONFIRMED (pow.cpp:47, miner.cpp:693, validation.cpp:8385).
- **Pre-hash eps lottery interaction:** eps (mainnet 18 bits) is a cheap 2×SHA256 pre-filter
  that gates which nonces reach the matmul; it is ORTHOGONAL and benefits honest+attacker
  equally. It does NOT close the per-matmul 12.8× edge (`/tmp/rtE1/epsilon.py`).
- **Sub-cubic on A·B itself (Strassen/structure):** A,B are full-entropy per-entry oracle
  (full rank w.h.p.) ⇒ no structure-specific shortcut on the one-time AB; not needed — AB is
  already amortized to ~free. (Generic Strassen would further cut the one-time cost; irrelevant.)

## NEW / unmitigated findings
- **CRIT — per-nonce PoW work is O(n²r), not the claimed O(n³); ~12.8× cheaper mining on
  mainnet, undetectably.** repro: `cd /tmp/rtE1 && python3 optimal2.py` (digest equality +
  op-counts). An attacker implementing the global-low-rank amortized prover mines every block
  at ~1/12.8 the honest field cost at identical difficulty ⇒ dominates hashrate / mines at a
  fraction of honest energy cost ⇒ profit + ability to 51%/destabilize. Bit-identical blocks ⇒
  no consensus signal to detect or reject.
  proposed mitigation (must break the AB amortization — make the dense product
  nonce-dependent):
  1. **Make seeds depend on the nonce** so A,B (hence A·B) change every attempt: e.g.
     `seed_a = SHA256(prev||height||sigma||0)` with sigma per-nonce — then AB is O(n³) per
     attempt and not reusable. Cost to honest: FromSeed is n² cheap SHA256/nonce (negligible
     vs n³), and it actually RESTORES the intended Ω(n³)/attempt. (Cleanest fix; needs the
     digest/sigma circularity handled — fold a *separate* per-nonce nonce field into seeds,
     distinct from the digest input.) This also removes the "precompute A·B once per tip"
     ASIC/FPGA advantage entirely.
  2. **OR raise the noise rank r toward n/6** so O(n²r) ≈ O(n³): neutralizes at r≈85 for n=512
     (vs current r=8) — but that ~10×s the honest noise cost too and is a blunt, partial fix
     (still a constant-factor edge, and bloats verify). Mitigation #1 is strictly better.
  3. **OR bind the dense product into the work non-linearly** (e.g. hash-chain the per-(i,j,ell)
     intermediate dense blocks so the transcript is NOT a linear function of the block products)
     — the linearity of `Compress` is what makes AB separable; a non-linear per-intermediate
     commitment (as the *legacy* prefix transcript partially did) raises the floor. But legacy
     still amortized to 5×, so non-linearity alone is insufficient; combine with #1.

## Refuted / robust (so others don't repeat)
- Naive `ReplayCanonicalHashWithReusableCleanProducts` reuse is a LOSS (3× slower) — do not
  conclude from the shipped `matmul_reuse_bench` that amortization is harmless; it measures the
  wrong decomposition. The damaging amortization is the GLOBAL low-rank one (this finding).
- No structure-specific sub-cubic shortcut on A·B is needed or found (A,B full-rank); the break
  is amortization, not a faster single product.

(/tmp/rtE1/{field_model,optimal2,cost_model,legacy,epsilon}.py); digest *equality* with the
