# b2 — Verification-asymmetry DoS & out-of-bounds / malformed proposed-block fields (B2 + parts B3/E2)

Scope: cheap-to-make blocks/headers that are expensive or fatal for honest verifiers; OOB/malformed
adversarially-settable fields. Empirical via `submitblock` (RPC) against honest nodes
`btx-node:pr214` (non-strict regtest, `fSkipMatMulValidation=true`) and the same image with
`-test=matmulstrict` (seed-enforcement + full phase2 active). Namespace `rtB2-`, /tmp/rtB2/.
Regtest consensus: matmul_dim pinned = **64**, transcript_block_size = 8, noise_rank = 4,
Freivalds rounds = 2, product-digest active @ h0, pre-hash epsilon = 0, powLimit `207fffff`.

Method: capture a freshly-mined valid block's raw hex, byte-patch individual fields, resubmit.
Patching any header field changes the block hash so the node runs full CheckBlock; patching only
matrix payload bytes keeps the hash → node short-circuits as `duplicate` (payload is not
header-committed). To force fresh-hash processing of a payload, I also bumped `nNonce64`.

## Vectors attempted

- **matmul_dim extremes (0,1,8,63,65,128,512,1024,2048,4096,65535)**: byte-patch offset 116 (uint16).
  RESULT: REJECTED(`high-hash`) for ALL, ~80–140ms each (RPC-dominated; actual validation sub-ms).
  Root cause: `CheckMatMulProofOfWork_Phase1` (src/pow.cpp:2015) requires
  `matmul_dim == params.nMatMulDimension` EXACTLY, evaluated in `CheckBlockHeader`
  (validation.cpp:7995) BEFORE any matrix allocation. No O(n²)/O(n³) work or allocation is reached.
- **nBits boundary (0, 0x80000001 neg-flag, 0x20800000 over-powLimit, 0xff000001, 0x22ffffff,
  0x1d00ffff, 0xffffffff)**: REJECTED(`high-hash`) for ALL, cheap. `DeriveTarget` (pow.cpp:1979)
  returns `{}` on negative/zero/overflow/over-powLimit → Phase1 fails before work.
- **matmul_digest = all-zero**: passes Phase1 (`0 ≤ target`) but REJECTED(`invalid-product-payload`)
  — product-committed digest doesn't match the C' payload. all-ff: `high-hash` (>target). Cheap.
- **seed_a/seed_b null / equal / corrupted** (non-strict): null→`high-hash`, equal→`invalid-product-payload`.
  (strict node, seed enforcement ON): seed_a corrupted→`bad-matmul-seeds`, seed_b=null→`high-hash`,
  seed_a==seed_b→`bad-matmul-seeds`. ContextualCheckBlockHeader (validation.cpp:8384) enforces
  `seed_{a,b} == DeterministicMatMulSeed(hashPrevBlock,height,{0,1})`; seeds are NOT attacker-free.
- **nNonce64 = garbage**: REJECTED(`invalid-product-payload`) — sigma changes → noise/product mismatch. Cheap.
- **dual nonce nNonce64 vs nNonce**: NON-ISSUE. Only `nNonce64` is serialized in the 182-byte BTX
  header (block.h:53 SERIALIZE_METHODS). `nNonce` is a memory-only legacy field, never on the wire
  or in the hash — no mismatch is expressible by a peer/submitter.
- **nVersion (0, 0x80000000, 0x7fffffff)**: 0→`bad-version`, neg→`bad-version`, max→`invalid-product-payload`. Cheap.
- **Freivalds product payload — oversized csize, short data** (csize 2M / MAX_SIZE / >MAX_SIZE words,
  few bytes): REJECTED(`Block decode failed`) at deserialization — chunked allocation
  (serialize.h MAX_VECTOR_ALLOCATE) + EOF, and `ReadCompactSize` caps at MAX_SIZE (0x02000000).
  No pre-allocation of a huge buffer from a bogus length prefix.
- **Freivalds payload — oversized BUT fully present** (1M words / 4MB; 3.75M words / 15MB; new-hash):
  REJECTED(`invalid-product-payload`). `IsMatMulFreivaldsPayloadSizeValid` (pow.cpp:2160) requires
  `matrix_c_data.size() == n*n == 4096`; any other shape is rejected by the shape check WITHOUT
  running Freivalds. The ~1.2s wall time is hex-decode of the 30MB string, not consensus work.
- **Freivalds payload — non-canonical values** (4096 words all 0xffffffff ≥ MODULUS): when only
  payload changes → `duplicate` (hash unchanged); with fresh hash → rejected. Per-element
  `>= matmul::field::MODULUS` check (pow.cpp:2208) rejects non-canonical entries inside Verify.
- **Freivalds payload — correctly-shaped but corrupted C'** (worst-case valid-shape reject; forces
  full A'/B' reconstruct + 2 Freivalds rounds, then fail): REJECTED(`invalid-product-payload`).
  Telemetry: mean **1.3ms**, max **3.8ms** validation per block (n=64). This is the most expensive
  malformed-block rejection path and it is still trivially cheap.
- **undersized payload** (declare 4096, give 2048 words): REJECTED(`Block decode failed`) at deser (EOF).
- **block weight / serialized-size @ 24M boundary (E2)**: matrix payload vectors ARE counted toward
  the consensus 24MB cap (CheckBlock validation.cpp:8175-8180 serializes the full CBlock incl.
  matrix_a/b/c via `GetSerializeSize(TX_WITH_WITNESS(block))`; block.h CBlock::SERIALIZE_METHODS
  writes the payload vectors for non-empty vtx). The 16MB↔24MB gap is NOT exploitable: P2P caps a
  single message at `MAX_PROTOCOL_MESSAGE_LENGTH = 16,000,000` (net.cpp:788) and the RPC HTTP body
  at `MAX_SIZE = 0x02000000 ≈ 32MB hex = 16MB block` (httpserver.cpp:573) — both stricter than the
  24MB consensus cap, so a payload-padded block can't reach the consensus size check oversized.
  Confirmed: 15MB and 16.1MB blocks deserialize and reach consensus (rejected `invalid-product-payload`);
  25MB/32MB hex > RPC limit → connection dropped at HTTP layer (NOT a crash; node stayed healthy).

## NEW / unmitigated findings
- (none — no accepted-when-should-reject, no crash, no memory blowup, no DoS amplification found.)

## DoS amplification quantification
- Flood of 50 worst-case valid-shape corrupted-C blocks: attacker ~91ms/block (RPC), verifier
  **1.3ms mean / 3.8ms max** of actual validation. Amplification ratio **< 1** (defender-favorable):
  the honest verifier does strictly LESS work than the attacker spends. No asymmetry exists because
  every expensive code path (SharedFromSeed, noise::Generate, Matrix alloc, Freivalds O(n²)) is gated
  behind the exact-dim Phase1 check that pins n to the consensus constant.
- On regtest the powLimit target is trivial, so a Phase1-valid header is free to make — but the
  reachable work is only the bounded O(n²) Freivalds (~1.3ms), so even free Phase1 headers yield no
  amplification. On mainnet a Phase1-valid header requires real PoW (digest≤target + epsilon gate),
  so the cheap-header→expensive-verify gap is also absent there.

## Refuted / robust (so others don't repeat)
- **Pathological matmul_dim (0/1/non-multiple/huge/overflow) forcing big alloc or O(n³)** — robust.
  Phase1 `matmul_dim != nMatMulDimension` exact check rejects before ANY allocation (pow.cpp:2015).
  matmul_dim is uint16 (max 65535), n*n*4 ≤ 16.7GB nominal but never reachable; min/max(64/2048)
  and `% transcript_block_size` and `noise_rank ≤ dim` checks are belt-and-suspenders behind it.
- **Oversized/undersized/wrong-shape/non-canonical Freivalds C' payload** — robust. Shape check
  `size()==n*n` and per-element `< MODULUS` reject before/within bounded O(n²) work; deserialization
  chunk-allocates and caps at MAX_SIZE; no memory blowup.
- **seed_a/seed_b null/equal/crafted** — robust. Deterministically derived & enforced in
  ContextualCheckBlockHeader (strict); null/inconsistent seeds also fail Phase1/product on non-strict.
- **dual nonce nNonce64 vs nNonce mismatch** — non-existent: nNonce is not serialized.
- **nBits at/over powLimit / zero / negative-encoded** — robust (DeriveTarget range check).
- **block-weight 24M bypass via matrix payload (E2)** — robust: payload counts toward the cap and
  transport limits (16MB P2P / 16MB-block RPC) are stricter than the 24MB consensus cap.
- **assert()/UB crash** — matrix.cpp/noise.cpp asserts (`row<m_rows`, `rows*cols≤UINT32_MAX`) are NOT
  adversary-reachable: every matrix in validation is `Matrix(n,n)` with n=64 pinned and loops bounded
  by the same n. No crash observed across the full battery; both nodes stayed healthy and advanced.

## Open questions / leads for next wave
- The matrix payload is parsed/hex-decoded (up to the ~16MB transport cap) BEFORE the cheap header
  dim check rejects. The work is bounded (≤16MB memcpy/decode, transient) and symmetric with the
  attacker's send cost, so not an amplification — but a deserialization-side early bound (cap
  matrix_c_data length to `nMatMulMaxDimension² ` words during read) would remove even the transient
  16MB allocation for junk-payload spam. Low priority / hardening nicety, not a vulnerability.
- B1 (Freivalds false-positive forgery) and B3 (pre-hash sigma shortcut) are the real soundness
  questions and are owned by b1/b3 — out of this finding's scope; nothing here contradicts them.
- Recommend a unit/fuzz test asserting CheckMatMulProofOfWork_Phase1 rejects before any
  matmul::Matrix construction (lock in the early-bound invariant against future refactors).
